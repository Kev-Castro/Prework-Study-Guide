HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • "h1","h2","h3","h4","h5","h6": Represnt the level of heading a given block represents.
  • "h1" *alone*: Is the opening tag
  • "/h1": Is the closing tag
  • "p" element: Is to represent a paragraph or block of text
  • "ul": Represents unordered list *must go first before li or ol*
  • "li": Reprsents listed items
  • "ol": Represents ordered lists
  • "img": Element that contains info about images that are displayed
  • "src": Is an attribute from within IMG and defines the location of the imagine
  • "br": Element that creates a line of empty space, or a line break, between two blocks of content

CSS

  • Cascading Style Sheets (CSS), which is a programming language used by developers to define how webpage content is presented to users.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The "box-shadow" CSS property adds shadow effects around an element's frame
  • The "border" CSS proprety adds a border around an element
  • The "margin" CSS property adds a margin top and bottom and "auto" left and right
  • "p" is used to align location of text, also to change size.
  • "ul" is used to declare font size of text under header and padding to either left or right
  • "img"is used to assign property values to any image element in the markup
  • The "display" property is assigned with the "blocK" value, which assigns each image to start on a new line
  • "text align" is one way to center a title or heading on a webpage
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • ***There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet***
  • When a developer wants to style an element directly in the HTML file, they use "inline CSS"
  • "Internal CSS style sheet" contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code
  • When using an "external CSS style sheet", we link the file using a "link" element placed in the HTML's "head"

Random Notes

  • Integrated development environment, or IDE: Is a type of software used by developers that combines many of the features needed by developers to do their work in a single interface, known as a graphical user interface, or GUI.
  • GUI: Is a way for users to interact with the computer visually.
  • MVP: Minimum Viable Product
  • Repository (“Repo”): Every project on github is called a repository.
  • Branch: Workspace where you can make changes that don't affect the live site.
  • When creating a new Branch name it to best describe the work. (Example: "contents-3-26") and make the description very detailed but short.
  • Markdown (".md"): The way content files are formatted in Github. It is also a way to write for the web using plain text; easily converts to HTML.
  • Commit Changes: Is saving a new version of a file you”re working on. Whenever you add, edit, or delete a file, you are making a commit. Commits are when you add change to your Branch.
  • Pull Request (“PR”): Asking for changes on a Branch to merge into another Branch. A “PR” is a way to discuss and review your work with others.
  • When requesting a PR, be as detailed as possible so your team can know what they're about to be looking at before fully looking into your changes.
  • Issue - ways to report new tasks or content fixes. Issues also track progress on Project Board
  • User Story - an explanation of a software feature from the user's perspective.
  • Acceptance Criteria - allows us to set the requirements that have to be met for us to complete our user story. Sometimes referred to as the definition of “done”.
  • Do Not Repeat Yourself (DRY) is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained. DRY can greatly reduce the overall size and complexity of a codebase.
